# ratproto-oauth Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build an AT Protocol OAuth 2.0 client crate implementing Authorization Code with PKCE, DPoP, and PAR.

**Architecture:** New `crates/ratproto-oauth` crate with ~10 focused source files. Uses `ratproto-crypto` for P-256 key generation and ES256 signing, `ratproto-identity` for DID resolution during issuer verification. JWTs built manually (no JWT crate). Storage traits for session/state persistence with in-memory implementations.

**Tech Stack:** ratproto-crypto (P-256/ES256), ratproto-identity (DID resolution), reqwest (HTTP), serde/serde_json, sha2, data-encoding (base64url), async-trait, p256 (point decompression for JWK), tokio, url.

```rust
use sha2::{Digest, Sha256};

/// PKCE (Proof Key for Code Exchange) challenge.
#[derive(Debug, Clone)]
pub struct PkceChallenge {
    /// 43-character base64url string (32 random bytes encoded).
    pub verifier: String,
    /// base64url(SHA-256(verifier)).
    pub challenge: String,
    /// Always "S256".
    pub method: &'static str,
}

/// Generate a PKCE challenge with a cryptographically random verifier.
pub fn generate_pkce() -> PkceChallenge {
    let mut bytes = [0u8; 32];
    rand::fill(&mut bytes);
    let verifier = base64url_encode(&bytes);
    let hash = Sha256::digest(verifier.as_bytes());
    let challenge = base64url_encode(&hash);
    PkceChallenge {
        verifier,
        challenge,
        method: "S256",
    }
}

/// Base64url encode without padding (RFC 4648 §5).
pub(crate) fn base64url_encode(data: &[u8]) -> String {
    data_encoding::BASE64URL_NOPAD.encode(data)
}

/// Base64url decode without padding.
pub(crate) fn base64url_decode(s: &str) -> Result<Vec<u8>, crate::OAuthError> {
    data_encoding::BASE64URL_NOPAD
        .decode(s.as_bytes())
        .map_err(|e| crate::OAuthError::Crypto(format!("base64url decode: {e}")))
}

#[cfg(test)]
#[allow(clippy::unwrap_used, clippy::expect_used, clippy::panic)]
mod tests {
    use super::*;

    #[test]
    fn pkce_verifier_length() {
        let pkce = generate_pkce();
        assert_eq!(pkce.verifier.len(), 43);
        assert_eq!(pkce.method, "S256");
    }

    #[test]
    fn pkce_challenge_is_sha256_of_verifier() {
        let pkce = generate_pkce();
        let hash = Sha256::digest(pkce.verifier.as_bytes());
        let expected = base64url_encode(&hash);
        assert_eq!(pkce.challenge, expected);
    }

    #[test]
    fn pkce_different_each_time() {
        let a = generate_pkce();
        let b = generate_pkce();
        assert_ne!(a.verifier, b.verifier);
    }
}
```

- [ ] **Step 5: Create `jwk.rs`**

```rust
use crate::OAuthError;

/// Encode a P-256 public key as a JWK JSON value for DPoP proof headers.
///
/// Decompresses the 33-byte SEC1 compressed point to extract the
/// 32-byte X and Y coordinates.
pub fn p256_public_jwk(compressed_bytes: &[u8; 33]) -> Result<serde_json::Value, OAuthError> {
    use p256::elliptic_curve::sec1::FromEncodedPoint;

    let encoded_point = p256::EncodedPoint::from_bytes(compressed_bytes)
        .map_err(|e| OAuthError::Crypto(format!("invalid SEC1 point: {e}")))?;

    // Decompress to get uncompressed point with x and y
    let point = p256::AffinePoint::from_encoded_point(&encoded_point);
    if point.is_none().into() {
        return Err(OAuthError::Crypto("point not on P-256 curve".into()));
    }
    let uncompressed = p256::EncodedPoint::from(point.unwrap());

    let x = uncompressed
        .x()
        .ok_or_else(|| OAuthError::Crypto("missing x coordinate".into()))?;
    let y = uncompressed
        .y()
        .ok_or_else(|| OAuthError::Crypto("missing y coordinate".into()))?;

    Ok(serde_json::json!({
        "kty": "EC",
        "crv": "P-256",
        "x": crate::pkce::base64url_encode(x),
        "y": crate::pkce::base64url_encode(y),
    }))
}

#[cfg(test)]
#[allow(clippy::unwrap_used, clippy::expect_used, clippy::panic)]
mod tests {
    use super::*;
    use ratproto_crypto::P256SigningKey;

    #[test]
    fn jwk_from_generated_key() {
        let sk = P256SigningKey::generate();
        let compressed = sk.public_key().to_bytes();
        let jwk = p256_public_jwk(&compressed).unwrap();

        assert_eq!(jwk["kty"], "EC");
        assert_eq!(jwk["crv"], "P-256");
        // x and y should be 43-char base64url strings (32 bytes encoded)
        assert_eq!(jwk["x"].as_str().unwrap().len(), 43);
        assert_eq!(jwk["y"].as_str().unwrap().len(), 43);
    }

    #[test]
    fn jwk_roundtrip_coordinates() {
        let sk = P256SigningKey::generate();
        let compressed = sk.public_key().to_bytes();
        let jwk = p256_public_jwk(&compressed).unwrap();

        // Decode x and y back to bytes
        let x_bytes = crate::pkce::base64url_decode(jwk["x"].as_str().unwrap()).unwrap();
        let y_bytes = crate::pkce::base64url_decode(jwk["y"].as_str().unwrap()).unwrap();
        assert_eq!(x_bytes.len(), 32);
        assert_eq!(y_bytes.len(), 32);

        // Reconstruct uncompressed point and re-compress
        let mut uncompressed = vec![0x04];
        uncompressed.extend_from_slice(&x_bytes);
        uncompressed.extend_from_slice(&y_bytes);
        let point = p256::EncodedPoint::from_bytes(&uncompressed).unwrap();
        let recompressed = point.compress();
        assert_eq!(recompressed.as_bytes(), &compressed[..]);
    }
}
```

- [ ] **Step 1: Implement DPoP module**

`dpop.rs` must provide:
- `create_dpop_proof(key: &P256SigningKey, method: &str, url: &str, nonce: Option<&str>, access_token: Option<&str>) -> Result<String, OAuthError>` — builds and signs a DPoP JWT
- `NonceStore` — thread-safe `HashMap<String, String>` keyed by origin

**DPoP JWT construction:**
1. Build header JSON: `{"alg":"ES256","typ":"dpop+jwt","jwk":{...}}`
2. Build payload JSON: `{"jti":"...","htm":"...","htu":"...","iat":...}` with optional `nonce` and `ath`
3. `jti`: base64url(16 random bytes)
4. `htu`: URL with query/fragment stripped
5. `ath`: base64url(SHA-256(access_token)) when access_token is Some
6. Encode: `base64url(header).base64url(payload)`
7. Sign with `key.sign(message_bytes)` — this does SHA-256 + ECDSA internally, producing ES256
8. Result: `base64url(header).base64url(payload).base64url(signature_64_bytes)`

```rust
    verifier: &str,
    redirect_uri: &str,
    auth: &dyn ClientAuth,
    dpop_key: &P256SigningKey,
    nonces: &NonceStore,
) -> Result<TokenSet, OAuthError>
```

This function:
1. Builds form params: `grant_type=authorization_code`, `code`, `code_verifier`, `redirect_uri`
2. Applies client auth
3. Creates DPoP proof (with nonce from store)
4. POSTs to token endpoint with `DPoP` header
5. If response is 400 with `use_dpop_nonce` error: update nonce store, retry once
6. Updates nonce store from `DPoP-Nonce` response header
7. Parses token response JSON
8. Validates: `access_token` present, `sub` present, scope contains "atproto", token_type is "DPoP" or empty
9. Returns TokenSet

**OAuthClient::new(config)** — constructs client, builds `ClientAuth` based on whether signing_key is provided.

**OAuthClient::authorize(opts) → Result<AuthorizeResult>:**
1. Resolve input to DID via `ratproto_identity::Directory`
2. Resolve DID to PDS endpoint
3. Fetch protected resource metadata from PDS
4. Fetch AS metadata from authorization server
5. Validate AS metadata
6. Generate DPoP key (P-256)
7. Generate PKCE challenge
8. Generate state parameter (16 random bytes, base64url)
9. Store AuthState
10. Send PAR request with DPoP proof + client auth
11. Handle nonce retry
12. Build authorization URL with `client_id` and `request_uri`
13. Return URL + state

1. Retrieve and delete state (one-time use)
2. Verify `iss` matches expected issuer
3. Exchange code for tokens (with DPoP key from state)
4. Verify issuer: resolve `sub` DID → PDS → AS matches
5. Delete any existing session for this user
6. Store and return new session

1. Load session from store
2. Check if token is stale → refresh if needed
3. Return AuthenticatedClient wrapping the session

**OAuthClient::sign_out(did) → Result<()>:**
1. Load session
2. Revoke token (best-effort)
3. Delete session

### Task 8: Authenticated Transport

**Files:**
- Create: `crates/ratproto-oauth/src/transport.rs`

- [ ] **Step 1: Implement AuthenticatedClient**

```rust
/// HTTP client with DPoP authentication and automatic token refresh.
///
/// Adds `Authorization: DPoP {access_token}` and `DPoP: {proof}` headers
/// to every request. Automatically refreshes stale tokens and retries on
/// DPoP nonce errors.
pub struct AuthenticatedClient {
    http: reqwest::Client,
    host: String,
    dpop_key: P256SigningKey,
    token_set: tokio::sync::RwLock<TokenSet>,
    auth: Arc<dyn ClientAuth>,
    nonces: Arc<NonceStore>,
    refresh_mutex: tokio::sync::Mutex<()>,
    session_store: Arc<dyn SessionStore>,
    did: String,
}
```

**Methods matching xrpc::Client interface:**

```rust
impl AuthenticatedClient {
    /// XRPC query (GET).
    pub async fn query<P: Serialize, O: DeserializeOwned>(
        &self, nsid: &str, params: &P
    ) -> Result<O, OAuthError>

    /// XRPC procedure (POST with JSON body).
    pub async fn procedure<I: Serialize, O: DeserializeOwned>(
        &self, nsid: &str, input: &I
    ) -> Result<O, OAuthError>
}
```

**Internal flow for each request:**
1. Check if token is stale → refresh (with mutex)
2. Read current access token
3. Create DPoP proof with `ath` (access token hash)
4. Send request with `Authorization: DPoP {token}` and `DPoP: {proof}` headers
5. Update nonce store from response `DPoP-Nonce` header
6. If 401 with `use_dpop_nonce`: retry with new nonce
7. If 401 with `invalid_token`: refresh token and retry once

**Refresh logic:**
1. Acquire refresh mutex
2. Re-check staleness (another request may have refreshed)
3. Call `refresh_token()`
4. Validate `sub` matches
5. Update stored token set and session store
6. Release mutex

## Notes for Implementers

- **Workspace clippy lints** deny `unwrap_used`, `expect_used`, `panic`, `unreachable`, `unsafe_code`. Use `?` with `.context()` or error conversion everywhere.
- **P-256 key access**: `P256VerifyingKey.inner` is private. For JWK coordinates, use the `p256` crate directly: `p256::PublicKey::from_sec1_bytes(&compressed_33_bytes)` → `to_encoded_point(false)` → `x()`, `y()`.
- **JWT signing**: `P256SigningKey::sign(content)` internally SHA-256 hashes then ECDSA signs — this IS ES256. The signature is 64 bytes R||S, which is the correct format for JWT ES256 signatures.
- **The XRPC client cannot be customized** with DPoP transport. `AuthenticatedClient` reimplements the query/procedure interface with DPoP headers rather than wrapping `xrpc::Client`.
- **`rand::fill(&mut bytes)`** is the correct API for `rand` 0.9+ to fill a byte array with random data.
- **Edition 2024**: This workspace uses Rust edition 2024. `async fn` in traits requires `async-trait` crate for dyn compatibility.
